hm

Write Graphviz dot files for archaeological sequence diagrams.

Install

The hm package is Common Lisp software that runs on SBCL and Clozure CL.

Installation with Quicklisp is recommended. It automatically installs dependencies.

cd ~/quicklisp/local-projects

git clone https://github.com/tsdye/harris-matrix.git

Load

(ql:quickload "hm")

Draw

(hm-draw "path/to/configuration/file")

Overview

The hm package produces Graphviz dot source code for archaeological sequence diagrams from tables of stratigraphic observations, inferences, and interpretations.

The hm package provides a single function, hm-draw, which expects a valid path to a configuration file as its sole argument.

The configuration file specifies paths to one or more data files and sets values for about four dozen Graphviz dot variables.

The configuration information and the data are held in comma separated value files. Together, the configuration and data files can hold a complete description of site stratigraphy and its interpretation.

Example Output

The stratigraphic section shown in Figure 12 of Edward Harris' book, Principles of Archaeological Stratigraphy. Note that the symbol for a deposit is a box and for an interface a trapezium. In this example, the nodes are colored according to the directed graph concept of level.

Data Tables

There are five data tables used as input to hm.

Units of Stratification

  • label: context identifier (primary key)
  • unit-type: interface or deposit
  • position: surface/basal/other
  • period: period identifier (foreign key)
  • phase: phase identifier (foreign key)
  • url: node link (svg output only)
label unit-type position period phase url

Observed Stratigraphic Relations

This table includes observations of stratigraphic superposition. Its first two columns refer to values from the label column of the contexts table.

  • younger: stratigraphically superior context label (foreign key)
  • older: stratigraphically inferior context label (foreign key)
  • url: arc link (svg output only)
younger older url

Inferred Parity Relations

This table contains inferences of parity between pairs of discontiguous contexts.

  • first: context label (foreign key)
  • second: context label (foreign key)
first second

Periods

  • id: a unique integer to identify the period (primary key)
  • label: name of the period used in the legend
  • color: zero-based integer scale for a Brewer color
  • description: optional field not used by hm
id label color description

Phases

  • id: a unique integer to identify the phase (primary key)
  • label: name of the phase used in the legend
  • color: zero-based integer scale for a Brewer color
  • description: optional field not used by hm
id label color description
